🚀 HOW IT WORKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SETUP PHASE:
   1. Run python3 setup_accounts.py
   2. Add User 1 with Zerodha/Dhan credentials
   3. Add User 2 with Zerodha/Dhan credentials
   4. (Repeat for more users)
   5. Run python3 verify_accounts.py to confirm

SCANNING PHASE:
   1. python3 scan_nifty50.py starts
   2. Loads all accounts from accounts.json
   3. Scans all 50 NIFTY stocks
   4. When signal found:
      - Sends alert for User 1 (with User 1 label)
      - Sends alert for User 2 (with User 2 label)
   5. Results saved to file

TELEGRAM ALERTS:
   Each user gets labeled alerts in same chat:

   🟢 SIGNAL DETECTED
   Stock: RELIANCE.NS
   Signal: BREAKOUT (BUY)
   Price: ₹2,850.50
   EMA Range: ₹2,800.00 - ₹2,900.00
   Account: Yogesh Kumar (yogesh)  ← USER LABEL

   (And separate alert for other users)

🔐 SECURITY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ Credentials are:
   - Stored locally in accounts.json
   - Protected by .gitignore (won't commit)
   - Never sent to external services
   - Never logged in console

✅ Best Practices:
   - Keep accounts.json private
   - Backup accounts.json securely
   - Rotate API keys periodically
   - Use unique credentials per account
   - Don't share the file

❌ Don't:
   - Commit accounts.json to git
   - Share credentials via chat/email
   - Log credentials
   - Use test credentials in production
