## Key Concepts

### User Account
- Unique ID (e.g., "yogesh", "john")
- Person's name
- Can have Zerodha credentials
- Can have Dhan credentials
- Or both

### Broker Credentials
- API Key
- Access Token
- User ID (on that broker)
- Stored in `accounts.json`

### Signal
- EMA breakout (BUY) or breakdown (SELL)
- Sent to all configured accounts
- Each alert labeled with account name

### Configuration
- All accounts in `accounts.json`
- Not committed to git
- Easy to backup

### Regular Tasks
- Rotate API keys (monthly)

- Backup `accounts.json`
- Note your credentials
- Test on dev first

### If Issues Arise
1. Run `verify_accounts.py`
2. Check relevant doc
3. Review `accounts.json`
4. Re-run setup script if needed
